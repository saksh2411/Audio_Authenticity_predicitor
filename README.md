## Overview

This repository contains an **Audio Deepfake Detection System** designed to classify speech audio as **Real**, **Fake**, or **Suspicious** using a hybrid pipeline that combines classical machine learning and deep learning. The project is built to address the growing risk of synthetic and manipulated speech generated by modern text-to-speech, voice conversion, and spoofing systems, which benchmark datasets such as ASVspoof were created to evaluate [1][2].

The implementation combines handcrafted acoustic feature extraction with CNN-based spectrogram learning, then integrates both prediction streams into a single inference pipeline for deployment. This hybrid design reflects the broader anti-spoofing practice of pairing engineered speech cues with learned representations for stronger robustness across attack types [1][3][4].

## Key Features

- Detects whether uploaded audio is authentic human speech or synthetic/manipulated speech.
- Uses a **CNN** for spectrogram-based deep feature learning.
- Uses **Random Forest** for classification on handcrafted acoustic features.
- Supports a three-way output: **Real**, **Fake**, and **Suspicious**.
- Includes a **Streamlit web app** for interactive testing and demonstration.
- Designed for academic presentation, viva, and practical deployment.

## Problem Statement

Recent progress in speech synthesis and voice conversion has made fake speech increasingly realistic, creating risks for voice biometrics, digital trust, social engineering, and misinformation [1][2]. Manual detection is unreliable for such attacks, so automated audio forensics systems are needed to identify subtle artifacts in synthetic speech [1][3].

This project aims to build a practical and presentation-ready system that can analyze an input speech sample and decide whether it is genuine or AI-generated by learning both acoustic statistics and time-frequency patterns.

## Datasets Used

### 1. ASVspoof 2019

ASVspoof 2019 is a standard anti-spoofing benchmark dataset containing spoofed and bona fide speech under **Logical Access (LA)** and **Physical Access (PA)** conditions [1][2]. The LA subset focuses on synthetic and converted speech attacks, while the PA subset targets replay-style attacks, making it highly useful for research on real-vs-fake speech detection [1][2].

**Purpose in this project:**
- Train and evaluate anti-spoofing models on research-standard data.
- Expose the system to multiple attack categories.
- Improve academic credibility by using a widely recognized benchmark [1].

### 2. FoR (Fake-or-Real) Dataset

The FoR dataset was introduced as a large-scale corpus for distinguishing real speech from synthetic speech, with more than 111,000 real utterances and more than 87,000 fake utterances [5][6]. It is useful for binary real-versus-synthetic classification and supports practical model training at larger scale [5][6].

**Purpose in this project:**
- Strengthen binary classification performance.
- Help the model learn broad differences between natural and generated speech.
- Provide more data variety for real-world deepfake detection experiments [5][6].

## Preprocessing

The audio pipeline first standardizes the input by loading speech in mono, normalizing amplitude, resampling to a fixed sample rate, and trimming or padding the signal to a fixed duration. Such fixed-shape preprocessing is commonly needed so feature extraction and CNN inference remain consistent across all input samples [7][4].

### Preprocessing steps

- Audio loading
- Mono conversion
- Amplitude normalization
- Resampling
- Fixed-length trimming/padding
- Spectrogram generation
- Handcrafted feature computation

## Features Used

### Handcrafted Acoustic Features

The classical ML branch uses engineered audio descriptors that capture spectral, temporal, and prosodic characteristics of speech. These include:

- MFCC mean and standard deviation
- Delta MFCC and Delta-Delta MFCC statistics
- Zero Crossing Rate (ZCR)
- Root Mean Square (RMS) energy
- Spectral centroid
- Spectral bandwidth
- Spectral rolloff
- Spectral flatness
- Spectral contrast
- Pitch statistics
- Silence ratio
- Signal-level summary statistics

MFCC-based features are widely used in speech analysis because they compactly represent perceptually relevant spectral information, while delta terms add temporal dynamics that help distinguish natural from synthetic speech behavior [3][7].

### Spectrogram Features

The deep learning branch uses **Mel Spectrogram / Log-Mel Spectrogram** style time-frequency representations as CNN input. This lets the model learn patterns directly from 2D speech representations, which is a common and effective approach for speech and audio classification [4][7].

## Algorithms Used

| Algorithm | Role in Project | Outcome |
|----------|------------------|---------|
| SVM | Classical baseline classifier | Around 93.0% accuracy in earlier project comparison [8] |
| Random Forest | Handcrafted-feature classifier | Around 97.0% accuracy in earlier comparison [8] |
| CNN | Deep spectrogram-based classifier | Around 98.5% accuracy in earlier comparison [8] |

### 1. Support Vector Machine (SVM)

SVM was used as a baseline machine learning classifier for comparison. It is a strong traditional model for high-dimensional classification tasks and helps establish whether more advanced models actually improve performance.

### 2. Random Forest

Random Forest is an ensemble method that combines many decision trees to improve predictive stability. In this project, it operates on handcrafted acoustic features and provides a strong ML baseline with good robustness to nonlinear relationships in the feature space.

### 3. Convolutional Neural Network (CNN)

CNN is the core deep learning model in the system and operates on spectrogram-like audio representations. It automatically learns local time-frequency patterns and hidden artifacts that may reveal synthetic or manipulated speech [4].

## Hybrid Model Strategy

The main strength of this project is the use of a **hybrid inference strategy**. Instead of depending only on one model, the system combines:

- A **Random Forest** branch for engineered acoustic features.
- A **CNN** branch for spectrogram-based deep pattern learning.
- A **score fusion layer** that merges both probabilities.

This improves reliability because handcrafted features capture known speech characteristics while CNNs discover abstract spectro-temporal artifacts from the data itself [3][4].

## Prediction Output

The final system produces one of the following labels:

- **Real** — the speech appears authentic.
- **Fake** — the speech is likely AI-generated or manipulated.
- **Suspicious** — the signal is uncertain or borderline, so the model avoids an overconfident binary decision.

The use of a “Suspicious” band makes the system more presentation-worthy and realistic because not every sample falls cleanly into a perfectly certain class.

## Model Performance

Earlier project results reported the following comparative accuracies:

- **SVM:** 93.0% [8]
- **Random Forest:** 97.0% [8]
- **CNN:** 98.5% [8]

These results show that the CNN performed best overall, while Random Forest also gave strong results and served as a valuable complement in the hybrid pipeline [8].
